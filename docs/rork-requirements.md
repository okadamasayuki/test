# iOSメモアプリ 要件定義書（rork用）

## 1. 概要

既存のWebメモアプリ（https://okadamasayuki.github.io/test/ ）の**iOSアプリ版**を作ってください。
メモとファイル置き場を持つ個人用アプリで、**既存のFirebaseバックエンドに接続し、Web版とiOSアプリの間でデータがリアルタイムに双方向同期される**ことが最重要要件です。

- UIはすべて日本語
- デザインの雰囲気: シンプル・白基調（ダークモード対応）、アクセントカラーは黄色(#ffcc00)、削除系は赤、同期状態は緑
- iPhone縦画面を主対象

## 2. 最重要制約: 既存バックエンドとの互換性

**既存のFirebaseプロジェクトをそのまま使います。新しいバックエンドを作らないでください。**
Web版が同じデータを読み書きするため、以下のデータ構造・規則を**厳密に**守ってください（これらの値は公開可能な識別子であり、秘密情報ではありません。アクセス制御はFirestoreセキュリティルールで行われています。ルールは設定済みなので変更不要です）。

### 2.1 Firebase接続設定

```js
{
  apiKey: "AIzaSyAuZEN_0GgN3JrVDE7aOSqRsAI2bkLYOsk",
  authDomain: "test-memo-c740a.firebaseapp.com",
  projectId: "test-memo-c740a",
  storageBucket: "test-memo-c740a.firebasestorage.app",
  messagingSenderId: "93414793974",
  appId: "1:93414793974:web:cb2cc17a732f2a1ba702d7",
}
```

- Firebase JS SDK（v10系）の `firebase/app`, `firebase/auth`, `firebase/firestore` を使用
- Cloud Storage / Cloud Functions は**使用しない**（無料プランのため）。ファイルも下記の通りFirestoreに保存する

### 2.2 Firestoreデータ構造（変更禁止）

すべてログイン中ユーザーのUID配下に置く:

**メモ: `users/{uid}/memos/{memoId}`**

```
{
  id: string,          // ドキュメントIDと同一。例: "mcr3k2a1b2c3d4"（英数字の短いランダム文字列）
  title: string,       // タイトル（空文字あり得る）
  body: string,        // 本文（空文字あり得る）
  createdAt: number,   // 作成時刻 (Unixミリ秒)
  updatedAt: number,   // 最終更新時刻 (Unixミリ秒)。同期の勝敗判定に使う
  order?: number,      // 手動並び順（0始まり昇順）。無いメモもある
  due?: number,        // 対応期日 (Unixミリ秒)。その日の23:59:59.999を指す。無ければ期日なし
  deleted?: true,      // 削除済みの墓標。trueの場合 title/body等は無い: {id, deleted:true, updatedAt}
  sample?: true        // 初回起動時に投入されたサンプルメモの印
}
```

**ファイルのメタデータ: `users/{uid}/files/{fileId}`**

```
{
  id: string,          // ドキュメントIDと同一
  name: string,        // ファイル名（日本語あり）
  size: number,        // バイト数
  type: string,        // MIMEタイプ (例 "image/jpeg", "application/pdf")
  chunkCount: number,  // 分割チャンク数
  createdAt: number    // Unixミリ秒
}
```

**ファイルの中身: `users/{uid}/chunks/{fileId}_{n}`**（n = 0,1,2,…）

```
{ data: string }   // ファイルをbase64化した文字列を700,000文字ごとに分割したもの
```

### 2.3 同期の規則（変更禁止）

- `users/{uid}/memos` と `users/{uid}/files` を `onSnapshot` でリアルタイム購読する
- **メモのマージはメモ単位のLast-Write-Wins**: 同じidのローカルとリモートは `updatedAt` が大きい方を採用
- スナップショット受信時、ローカルの方が新しい/リモートに無いメモは `setDoc` でアップロードする
- **メモの削除は物理削除ではなく墓標**: `{ id, deleted: true, updatedAt: 今 }` で上書きする。墓標は一覧に表示しない。60日より古い墓標はマージ時に破棄してよい
- メモの内容・期日・並び順を変更したら必ず `updatedAt` を現在時刻に更新して `setDoc`
- **ファイルの削除は物理削除**: metaドキュメントと全チャンクを `deleteDoc`
- ファイルのアップロードは**チャンクを全部書き込んでからmetaを最後に書く**（一覧には完成したファイルだけが載るように）
- ファイルサイズ上限は20MB

### 2.4 やってはいけないこと

- Firestoreのスキーマ変更、フィールド名変更、新しいコレクションの追加
- メモの物理削除（墓標方式を守る）
- Firebaseコンソール側の設定変更を要求すること（セキュリティルール・認証設定は設定済み）

## 3. 認証

Firebase Authを使用:

- **メールアドレス＋パスワードの新規登録／ログイン**（必須）
- パスワードリセットメール送信（必須）
- Googleログイン（**任意**。ネイティブでの実装が難しければ省略してよい。メール認証があれば十分）
- 認証エラーは日本語で表示（例: 「メールアドレスまたはパスワードが違います」「このメールアドレスは登録済みです」「パスワードは6文字以上にしてください」）
- **ログアウトすると端末内のメモのキャッシュを消去する**（クラウド側は残り、再ログインで復元される）
- 未ログインでも、端末内だけのメモ帳として使える（後述のローカル保存）

## 4. 画面仕様

### 4.1 全体構成

- 上部にボタン列: **☑（選択モード）／＋（新規メモ。ファイルタブでは⬆アップロード）／⚙（設定）**
  - ⚙は**同期が有効なとき緑背景・白アイコン**、同期エラー時は赤
- その下に**タブ: 「メモ」「ファイル」**（アプリ再起動後も最後に選んだタブを維持）
- タブの下に検索欄＋「期日順」ボタン（メモタブのみ）

### 4.2 メモ一覧

- 行の表示: タイトル（未入力なら「無題のメモ」）、本文1行目のプレビュー、更新日時（当日はHH:MM、他はYYYY/MM/DD）
- **期日チップ**: 期日ありのメモは日時の隣に表示
  - 期限切れ（期日 < 現在）: 赤の塗りつぶしで「期限切れ」
  - 本日中: オレンジの塗りつぶしで「本日中」
  - 明日中・今週中: オレンジの枠線
  - 来週中・それ以降: グレーの枠線で「M/Dまで」
- **並び順**: `order` があるメモは order 昇順。無いメモは更新が新しい順で先頭側
- **「期日順」ボタン**: 押した時だけ、期日ありメモを期日が近い順で先頭に並べ替え（結果を order に保存して同期）。期日を設定しただけでは並びは変わらない
- **検索**: タイトル・本文の部分一致で絞り込み
- **左スワイプで削除**: 行を左にスワイプすると赤い「削除」ボタンが現れる。タップで確認ダイアログ→削除。縦スクロールと干渉しないこと
- **ドラッグ＆ドロップ並び替え**: 行の右端のハンドル(⠿)を掴んで上下に移動。掴んだカードが浮き上がって指に追従し、他の行がアニメーションで場所を空ける（iOSネイティブの並び替えUIでも可）。確定した並びは order に保存して同期。検索中・選択モード中は並び替え不可

### 4.3 メモ編集画面（全画面）

- タイトル入力欄＋本文入力欄。**入力は自動保存**（約400msのデバウンス。保存時にupdatedAt更新→同期）
- **期日セレクタ**: なし／本日中／明日中／今週中／来週中／日付を指定（カレンダー）
  - 「今週中」は次の日曜日の終わり、「来週中」はその翌週の日曜日の終わり
  - 期日はその日の23:59:59.999のタイムスタンプとして保存
- 一覧に戻る: **画面の右スワイプ**、「← 一覧に戻る」ボタン
- 下部に「最終更新: …」表示と「削除」ボタン（確認ダイアログあり）
- **空メモの自動削除**: 新規作成後、タイトルも本文も空のまま（一度入力して消した場合も含む）他の画面へ移動したら、その新規メモは確認なしで自動削除する
- 他端末からの変更は編集中でもリアルタイム反映（編集中のカーソルを壊さないよう、値が変わった時のみ反映）

### 4.4 ファイルタブ

- 要ログイン（未ログイン時は「ログインしてください」の案内）
- **⬆アップロード**: 端末のファイル/写真を選択（複数可）。アップロード中は「⬆ ファイル名 … n%」の進捗行を表示
- 一覧の行: ファイル名、サイズ（KB/MB表記）＋「タップでプレビュー/ダウンロード」、日時
- **タップでプレビュー**（モーダル/シート）:
  - 画像（image/*）: 画像表示
  - PDF: PDF表示（iOSのネイティブビューア可）
  - テキスト/JSON: テキスト表示
  - Word(.docx): 可能なら本文プレビュー（**任意**。難しければダウンロードのみでよい）
  - その他（zip等）: プレビューせず直接ダウンロード
- プレビューには **「削除」（確認あり）と「ダウンロード」**（iOSでは共有シート/ファイルに保存）ボタン、右上に✕
- 左スワイプで削除（メモと同じ操作感）
- ダウンロードはチャンクを全部読んで結合→base64デコード→元ファイルとして保存/共有

### 4.5 一括削除（選択モード）

- ☑を押すと選択モード（☑は✕に変わる）。上部に「すべて選択」「n件選択中」「削除 (n)」のバーを表示
- 行タップでチェックの付け外し（○→✓）。「削除」で確認ダイアログ1回→まとめて削除
- メモ・ファイル両タブで使える。タブ切替やキャンセルで選択解除

### 4.6 設定画面（⚙）

- 未ログイン時: メール/パスワードの入力欄＋「新規登録」「ログイン」ボタン、パスワードリセットリンク
- ログイン中: 「〇〇としてログイン中」表示＋「ログアウト」ボタン
- 同期状態の表示（同期中=⚙緑。エラー時はメッセージ表示）

## 5. ローカル動作・オフライン

- メモは常に端末内（AsyncStorage等）にもキャッシュし、未ログイン・オフラインでも閲覧/編集できる。ログインすると差分がクラウドにアップロードされる
- **初回起動時（キャッシュが空のとき）だけ**、仕事関連のサンプルメモ10件をローカルに投入する（`sample: true` を付ける）。全削除しても復活しないこと
- ログイン時、**リモートに実データがあれば、ローカルの未編集サンプル（sample=true かつ updatedAt==createdAt）は破棄**する（重複防止）

## 6. 任意（できれば）項目 — 実装が難しければ省略可

- Googleログイン
- docxのプレビュー
- ドラッグ並び替えの「浮き上がり」アニメーションの再現（標準の並び替えUIで代替可）
- サンプルデータ投入ボタン（設定画面から、サンプルのメモ・ファイルを一括投入する機能）

## 7. 受け入れ基準

Web版（https://okadamasayuki.github.io/test/ ）と同じアカウントでログインした状態で:

1. iOSで作成したメモが数秒以内にWeb版に表示される（逆方向も）
2. iOSでのメモ編集・削除・並び替え・期日設定がWeb版に反映される（逆方向も）
3. Web版でアップロードしたファイルがiOSの一覧に出て、プレビュー/ダウンロードできる
4. iOSでアップロードしたファイルがWeb版でプレビュー/ダウンロードできる
5. メールアドレスで新規登録・ログイン・ログアウトができ、ログアウトで端末内データが消え、再ログインで復元される
6. 未ログインでもメモの作成・編集ができ、初回起動時にサンプルメモ10件が入っている
7. 一覧の左スワイプ削除、選択モードでの一括削除が動く
8. 期日チップの色分け（期限切れ=赤、本日中=オレンジ）が正しく、「期日順」ボタンで期日が近い順に並ぶ
9. 空のまま放置した新規メモが自動で消える
10. 既存のWeb版のデータが壊れない（スキーマ変更をしていない）
