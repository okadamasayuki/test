# iOSアプリ版 要件定義書（Claude Code向け）

## 0. この文書について

あなた（Claude Code）への依頼は、既存のWebメモアプリの**iOSアプリ版**を新規に開発することです。

- **参照実装（仕様の正）**: リポジトリ `okadamasayuki/test` のWebアプリ
  - 公開URL: https://okadamasayuki.github.io/test/
  - 本体は `index.html` / `style.css` / `app.js`（ビルド不要のバニラJS、約1200行）
  - この文書と実装が食い違う場合や詳細が曖昧な場合は、**`app.js` を読んで挙動を合わせてください**
- 最重要要件は **既存Firebaseバックエンドとのデータ互換**（§2）。Web版とiOSアプリで同じアカウントのメモ・ファイルがリアルタイムに双方向同期されること

## 1. 技術方針

- **Expo (React Native) + TypeScript** を推奨します。理由:
  - Mac/XcodeなしでもExpo Go（iPhoneの実機アプリ）で開発・動作確認できる
  - Firebase JS SDK（Web版と同じもの）がそのまま使え、同期ロジックをWeb版から素直に移植できる
- Firebaseは **Firebase JS SDK（v10系以降）** の `firebase/app` / `firebase/auth` / `firebase/firestore` を使用
  - React NativeでのAuth永続化は `initializeAuth` + `getReactNativePersistence`（AsyncStorage）を使うこと
- Cloud Storage / Cloud Functions は**使わない**（無料Sparkプランのため）。ファイルは§2.2の通りFirestoreに保存
- ローカルキャッシュは AsyncStorage（Web版のlocalStorage相当）
- プロジェクトは**新規ディレクトリ/リポジトリ**として作成する。Web版のファイルは変更しない

## 2. 最重要制約: 既存バックエンドとのデータ互換

以下は**変更禁止**です。Web版が同じデータを読み書きします。

### 2.1 Firebase接続設定

Web版の `firebase-config.js` にある値をそのまま使ってください（プロジェクト `test-memo-c740a`）。
この設定値は公開可能な識別子であり秘密情報ではありません。アクセス制御はFirestoreセキュリティルール
（`users/{userId}/{document=**}` は本人のみ読み書き可）で行われており、**設定済み・変更不要**です。

### 2.2 Firestoreデータ構造

**メモ: `users/{uid}/memos/{memoId}`**

```
{
  id: string,          // ドキュメントIDと同一。Web版は Date.now().toString(36)+乱数6文字 で生成
  title: string,       // 空文字あり得る
  body: string,        // 空文字あり得る
  createdAt: number,   // Unixミリ秒
  updatedAt: number,   // Unixミリ秒。同期の勝敗判定に使う
  order?: number,      // 手動並び順（0始まり昇順）。無いメモもある
  due?: number,        // 対応期日。その日の23:59:59.999のUnixミリ秒。無ければ期日なし
  deleted?: true,      // 削除済みの墓標。この場合の形は {id, deleted:true, updatedAt} のみ
  sample?: true        // 初回シードされたサンプルメモの印
}
```

**ファイルmeta: `users/{uid}/files/{fileId}`** — `{ id, name, size, type, chunkCount, createdAt }`

**ファイル本体: `users/{uid}/chunks/{fileId}_{n}`**（n=0,1,2,…） — `{ data: string }`
ファイルはbase64文字列にして **700,000文字ごとに分割**して保存。上限20MB。

### 2.3 同期規則（`app.js` の applySnapshot / mergeMemos / pushMemo を移植）

- `memos` / `files` コレクションを `onSnapshot` でリアルタイム購読
- メモのマージは**メモ単位のLast-Write-Wins**（`updatedAt` が大きい方を採用）
- スナップショット受信時、ローカルが新しい/リモートに無いメモは `setDoc` で書き戻す
- **メモの削除は墓標方式**: `{id, deleted:true, updatedAt:今}` で上書き。一覧非表示。60日超の墓標はマージ時に破棄可
- メモの内容・期日・並び順の変更時は必ず `updatedAt` を更新して `setDoc`
- **ファイルの削除は物理削除**（meta＋全チャンクを `deleteDoc`）
- ファイルはチャンク全書き込み後にmetaを最後に書く
- ログイン時、リモートに実データがあれば、ローカルの未編集サンプル（`sample:true` かつ `updatedAt===createdAt`）は破棄（サンプル重複防止）

## 3. 機能要件

Web版の全機能を移植します。詳細挙動は `app.js` を参照。

### 3.1 認証（⚙設定画面）

- メール/パスワードの新規登録・ログイン・リセットメール送信（Firebase Auth）
- Googleログイン（**任意**。expo-auth-session等の追加設定が必要なら省略可。メール認証だけで受け入れ可）
- 認証エラーは日本語表示（auth/invalid-credential→「メールアドレスまたはパスワードが違います」等。`app.js` の `authErrMsg` 参照）
- **ログアウト時は端末内のメモキャッシュを消去**（クラウドは残り、再ログインで復元）
- 未ログインでも端末内だけのメモ帳として動作すること

### 3.2 画面構成

- ヘッダーのボタン列: **☑選択 ／ ＋新規（ファイルタブでは⬆アップロード）／ ⚙設定**
  - ⚙は同期中=緑背景・白、同期エラー=赤
- タブ「メモ」「ファイル」（最後に選んだタブをアプリ再起動後も維持）
- 検索欄＋「期日順」ボタン（メモタブのみ）

### 3.3 メモ一覧

- 行: タイトル（空なら「無題のメモ」）／本文1行目／更新日時（当日はHH:MM、他はYYYY/MM/DD）
- **期日チップ**: 期限切れ=赤塗り「期限切れ」、本日中=オレンジ塗り、明日中・今週中=オレンジ枠、来週中以降=グレー枠「M/Dまで」
- 並び順: `order` 昇順（orderが無いメモは更新が新しい順で先頭側）
- **「期日順」ボタン**: 押した時のみ、期日ありメモを期日昇順で先頭に整列（結果をorderに保存して同期）。期日を設定しただけでは並びは動かさない
- **左スワイプで削除**（確認ダイアログあり）。縦スクロールと干渉しないこと
- **ドラッグで並び替え**: 行右端のハンドル(⠿)を掴んで移動。掴んだカードが浮き上がって追従し、他の行がアニメーションで退避（react-native-draggable-flatlist等の利用可）。検索中・選択モード中は無効

### 3.4 メモ編集（全画面）

- タイトル・本文の**自動保存**（400ms程度のデバウンス）
- **期日セレクタ**: なし／本日中／明日中／今週中／来週中／日付指定（ネイティブのDatePicker）
  - 「今週中」=次の日曜の終わり、「来週中」=その翌週の日曜の終わり（`app.js` の `presetDue` 参照）
- 一覧へ戻る: 画面の右スワイプ＋「← 一覧に戻る」（iOS標準のバックジェスチャで可）
- **空メモの自動削除**: 新規作成後、タイトル・本文とも空のまま（一度入力して消した場合も含む）他画面へ移動したら確認なしで自動削除
- 他端末からの変更は編集中もリアルタイム反映（入力中のカーソルを壊さない）

### 3.5 ファイルタブ（要ログイン）

- ⬆アップロード: 端末のファイル/写真を選択（複数可）、進捗%を行として表示
- タップで**プレビュー**: 画像／PDF／テキスト・JSON はアプリ内表示。docxは任意（難しければダウンロードのみ）。zip等その他はダウンロードのみ
- プレビューに「削除」（確認あり）と「ダウンロード」（iOSの共有シート/ファイルに保存）、右上✕
- 左スワイプで削除。削除時はmetaとチャンクを両方消す

### 3.6 一括削除（選択モード）

- ☑で選択モード開始（ボタンは✕に）。行タップでチェック、上部バーに「すべて選択」「n件選択中」「削除(n)」
- 確認ダイアログ1回でまとめて削除。メモ・ファイル両対応。タブ切替/キャンセルで解除

### 3.7 初回サンプル

- 初回起動（キャッシュ空）時のみ、仕事関連サンプルメモ10件をローカル投入（`sample:true`。内容は `app.js` の `sampleMemos` を流用）。全削除後に復活しないこと

## 4. 非機能・スコープ外

- UIは日本語。ライト/ダークモード対応（アクセント黄 #ffcc00、削除系は赤、同期状態は緑）
- スコープ外: プッシュ通知、ウィジェット、App Store申請作業、Firebaseコンソール設定変更、Web版の変更

## 5. 開発の進め方（推奨マイルストーン）

検証はExpo Go（ユーザーのiPhone）で行う前提。各段階でユーザーに動作確認してもらってから次へ進むこと。

1. **M1: 骨格＋ローカルメモ** — タブ/一覧/編集/検索/サンプルシード（AsyncStorageのみ、Firebase未接続）
2. **M2: 認証＋メモ同期** — ログイン/登録、onSnapshot購読、マージ・墓標。**Web版と並べて双方向同期を確認**
3. **M3: メモUX** — スワイプ削除、ドラッグ並び替え、一括削除、期日（チップ・期日順・自動削除）
4. **M4: ファイル** — アップロード/一覧/プレビュー/ダウンロード/削除
5. **M5: 仕上げ** — ダークモード、エラーメッセージ、ログアウト時の消去、細部の磨き込み

同期ロジック（mergeMemos/墓標/サンプル破棄）は**純粋関数として切り出してユニットテストを書く**こと。

## 6. 受け入れ基準

Web版（https://okadamasayuki.github.io/test/ ）と同じアカウントでログインして:

1. iOSで作成/編集/削除したメモが数秒でWeb版に反映される（逆方向も）
2. 並び替え・期日の変更が双方向に同期される
3. Web版でアップロードしたファイルがiOSでプレビュー/保存できる（逆方向も）
4. メール新規登録→ログアウト（端末データ消去）→再ログイン（復元）が動く
5. 未ログインでもメモが使え、初回起動時にサンプル10件が入る
6. 左スワイプ削除・一括削除・ドラッグ並び替えが動く
7. 期日チップの色分けと「期日順」整列が仕様通り
8. 空のまま放置した新規メモが自動で消える
9. Web版のデータを壊していない（スキーマ変更なし・墓標方式の維持）
